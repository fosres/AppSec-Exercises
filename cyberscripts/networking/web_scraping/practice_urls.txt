# =============================================================================
# NOTES FOR YOUR RECONNAISSANCE REPORT
# =============================================================================
#
# When analyzing each site, document:
#
# 1. SERVER DISCLOSURE
#    - Does Server header reveal version? (e.g., "nginx/1.18.0")
#    - Is X-Powered-By present? (e.g., "PHP/7.4")
#    - Any other technology fingerprints?
#
# 2. SECURITY HEADERS (Score out of 7)
#    [ ] X-Frame-Options
#    [ ] X-Content-Type-Options
#    [ ] X-XSS-Protection
#    [ ] Strict-Transport-Security
#    [ ] Content-Security-Policy
#    [ ] Referrer-Policy
#    [ ] Permissions-Policy
#
# 3. COOKIE SECURITY
#    - Session cookies have HttpOnly?
#    - Session cookies have Secure?
#    - SameSite attribute set?
#
# 4. CORS CONFIGURATION
#    - Is Access-Control-Allow-Origin set?
#    - If *, is Allow-Credentials also true? (VULNERABILITY!)
#
# 5. INTERESTING FINDINGS
#    - Rate limiting headers?
#    - Custom headers revealing architecture?
#    - Cache configuration?
#
# =============================================================================
# REMEMBER: PASSIVE RECONNAISSANCE ONLY
# =============================================================================
# This exercise is about READING responses, not attacking.
# Never send:
# - SQL injection payloads
# - XSS attempts
# - Authentication brute force
# - Fuzzing requests
# - Any malicious content
#
# You are simply making normal GET requests and analyzing the response.
# This is equivalent to visiting the site in a browser and viewing headers.
